# Plan: Multi-Provider Image Generation Tool (`image_create`)

## Overview

Create a new `image_create` tool that uses the OpenAI images/generation API format, supporting any OpenAI-compatible provider (OpenAI, OpenRouter, custom endpoints). The tool reads provider/model config from `config.yaml` and supports automatic upscaling via Pillow.

## Changes

### 1. New file: `tools/image_tool.py`

Main tool implementation (~300 lines):

- **Config loading**: `_load_image_gen_config()` reads `image_gen:` section from `~/.hermes/config.yaml`
- **Provider resolution**: `_resolve_provider()` returns (provider, api_key, base_url, model) tuple
  - Provider options: `openai`, `openrouter`, `custom`
  - API key resolution per provider: `OPENAI_API_KEY`, `OPENROUTER_API_KEY`, `IMAGE_GEN_API_KEY`
  - Default model per provider: `dall-e-3`, `openai/dall-e-3`, or user-configured
- **Image generation**: `_generate_image()` uses `openai` Python SDK with `base_url` override
- **Upscaling**: `_upscale_image()` downloads image, upscales with Pillow (LANCZOS), saves to `get_hermes_home()/images/`
- **Main function**: `image_create_tool(prompt, aspect_ratio, size, model, quality)` returns JSON
- **Check function**: `check_image_create_requirements()` returns True if configured provider has API key
- **Schema**: OpenAI function schema with `prompt` (required), `aspect_ratio`, `size`, `model`, `quality` params
- **Registration**: `registry.register(name="image_create", toolset="image_create", ...)`

## Config Structure

```yaml
image_gen:
  provider: "openai"       # "openai" | "openrouter" | "custom"
  model: ""                # empty = provider default, or e.g. "dall-e-3", "openai/dall-e-3"
  base_url: ""             # empty = provider default, or e.g. "https://openrouter.ai/api/v1"
  upscale:
    enabled: true
    max_width: 2048
    max_height: 2048
```
